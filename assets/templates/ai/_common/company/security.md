# セキュリティガイドライン

## シークレット管理

- APIキー・パスワードは必ず環境変数で管理する
- `.env` は `.gitignore` に追加し、絶対にコミットしない
- `.env.example` にキー名のみ記載（値は空にする）
- シークレットを誤ってコミットした場合は即座にローテーションする

## 入力値の検証

- 外部からの入力は必ずバリデーションする
- SQL クエリはプレースホルダーを使用する（直接埋め込み禁止）
- HTML を出力する場合は必ずエスケープする

## 依存関係

- `npm audit` / `pip-audit` を定期実行する
- 既知の脆弱性がある依存を放置しない

## API セキュリティ

- 認証が必要なエンドポイントは必ず認証を実装する
- レートリミットを設ける
- CORS 設定を適切に行う（`*` は本番 NG）
