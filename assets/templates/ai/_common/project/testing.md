# テスト方針

## テストの種類

- **Unit Test** — 単一の関数・クラスをテストする
- **Integration Test** — 複数コンポーネントの連携をテストする

## テスト命名規則

```
describe('対象の名前', () => {
  it('〜すること', () => { ... })
  it('〜の場合に〜すること', () => { ... })
})
```

## テストカバレッジ目標

- 新規コード: 80% 以上
- ビジネスロジック: 100% を目指す

## テストを書かなくてよいもの

- UI コンポーネントの見た目
- 設定ファイル
- 型定義のみのファイル

## テストの禁止事項

- 実際の API・DB を叩くテストをデフォルトで実行しない（CI が壊れる）
- テスト間で状態を共有しない（各テストは独立させる）
- `console.log` をテストコードに残さない
