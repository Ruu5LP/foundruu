# Bugfix Workflow

バグ修正のフロー。原因特定を修正より先に行う。

## ステップ

1. **再現** — バグを再現する手順・入力・環境を特定する。再現できない場合は `questions.md` に確認事項を書く
2. **原因特定** — ログ・コードを追い、根本原因を特定する。症状への対処（パッチ）と根本修正を区別する
3. **影響範囲確認** — 同じ原因による他の不具合、修正による副作用を洗い出す
4. **失敗するテストを書く** — 修正前にバグを再現するテストを書き、失敗することを確認する
5. **修正** — 最小限の変更で根本原因を修正する。無関係なリファクタリングを混ぜない
6. **検証** — 追加したテストが通ること、既存テストが壊れていないことを確認する
7. **サマリ** — 原因 / 修正内容 / 影響範囲 / 再発防止策を `summary.md` に書く

## ルール

- 原因が特定できていない状態で修正を始めない
- 修正とリファクタリングを同じ変更に混ぜない
- 「なぜ既存テストで検出できなかったか」を振り返りに書く
