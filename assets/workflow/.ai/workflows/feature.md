# Feature Workflow

新機能を追加するときのフロー。Development Workflow をベースに、新機能特有の確認を加える。

## ステップ

1. **要件確認** — 何を作る / 作らないの境界、正常系・異常系、権限、完了条件を確認する。不足があれば `questions.md` へ
2. **既存調査** — 類似機能・共通化できる既存コード・影響を受ける既存仕様を洗い出す
3. **設計** — 新規/変更ファイル、API入出力、DBスキーマ変更（と理由）、UI、エラーケースを `design.md` に書く
4. **タスク分解** — 動作確認可能な単位に分割する
5. **実装** — 既存の規約・アーキテクチャに従う。新しい依存の追加は理由を明記する
6. **テスト** — 新機能のテストを追加し、既存テストが全て通ることを確認する
7. **ドキュメント** — README / API ドキュメント等、影響するドキュメントを更新する

## チェックリスト

- [ ] 既存機能への影響を設計に明記した
- [ ] 異常系・境界値のテストを書いた
- [ ] `.env.example` / 設定ファイルの追記漏れがない
- [ ] ドキュメントを更新した
