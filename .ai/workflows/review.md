# Review Workflow

コードレビュー（人間 / AI）のフロー。

## ステップ

1. **コンテキスト把握** — 要件（`requirements.md`）・設計（`design.md`）と差分を突き合わせる
2. **正しさ** — 要件を満たしているか、異常系・境界値・権限・競合状態に問題がないか
3. **影響範囲** — 既存機能・DBスキーマ・API互換性への影響が設計どおりか
4. **品質** — 既存の規約・アーキテクチャに沿っているか、重複・過剰な複雑さがないか
5. **テスト** — テストが観点（正常系 / 異常系 / 境界値 / 既存影響）を網羅しているか
6. **セキュリティ** — 入力検証、認証・認可、秘密情報のハードコード、インジェクション
7. **指摘** — 重要度順（blocker / should / nit）に、理由と修正案をセットで書く

## ルール

- 指摘には必ず「なぜ問題か」と「どう直すか」を添える
- 好みの問題（nit）と修正必須（blocker）を明確に区別する
- 差分に現れないが影響を受ける箇所（呼び出し元・設定・ドキュメント）も確認する
