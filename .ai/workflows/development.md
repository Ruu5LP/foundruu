# Development Workflow

AIエージェント（Claude Code / Codex 等）が開発を行うときの基本フロー。
詳細な要件ゲートのルールは `.ai/prompts/session-workflow.md` を参照すること。

## フロー

```
要件を読む → 実装可能か判定
  ├─ 不足 → questions.md に質問（重要度順・最大5件）→ 回答待ち
  └─ 十分 → 設計 → タスク分解 → 実装 → テスト → サマリ
```

## ステップ

1. **要件確認** — `requirements.md`（またはユーザーの指示）を読み、目的・対象範囲・完了条件が実装可能なレベルか判定する
2. **設計** — `design.md` に方針 / 変更対象 / DB / API / UI / 影響範囲を書く
3. **タスク分解** — `tasks.md` に実装タスク・確認タスク・完了条件を書く
4. **実装** — タスク単位で進め、既存コードの規約・スタイルに合わせる
5. **テスト** — `test.md` の観点（正常系 / 異常系 / 境界値 / 既存影響）に沿って確認し、自動テストを実行する
6. **サマリ** — `summary.md` に実施内容 / 変更ファイル / 残課題 / 振り返りを書く

## ルール

- 要件が曖昧なまま実装を始めない
- 大量の変更を一度に行わず、タスク単位でコミット可能な状態を保つ
- 破壊的変更・スキーマ変更・認証/決済まわりは必ず設計に明記する
