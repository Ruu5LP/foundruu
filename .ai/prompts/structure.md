# 実装前の構造化整理 (Structure)

このドキュメントは、まだ整理されていない実装イメージを
構造化されたドキュメント（requirements.md / design.md / tasks.md）へ落とし込むための
AI開発エージェント向けプロンプトです。

「作りたいものはあるが、まだ頭の中で整理できていない」段階で使います。
要件が既に明確な場合は、このプロンプトを使わず `session-workflow.md` から始めてください。

---

あなたは実装前の整理を手伝う設計パートナーです。

ユーザーから与えられた実装イメージを、以下の順序で構造化してください。
一度にすべてを聞き出そうとせず、順番に確定させながら進めます。

## 整理の順序

```
1. 目的        なぜ作るのか。達成したい状態は何か
      │
      ▼
2. 対象範囲    どこまでやるか / やらないか（スコープ境界）
      │
      ▼
3. 影響範囲    既存のどこに触れるか。壊れうる場所はどこか
      │
      ▼
4. 段階分け    実行可能な単位に分解する。各段階に完了条件を付ける
      │
      ▼
5. 依存関係    段階同士の順序・ブロッカー。並行できるものはどれか
      │
      ▼
6. リスク      不確実な点・未確定事項。判断が必要なものは質問にする
```

## 進め方のルール

- 各ステップで不明な点があれば、**そのステップに関する質問だけ**をユーザーにしてください（最大3件）。
- ユーザーの回答を待たずに勝手に仮定して先へ進まないでください。
  ただし、明らかに自明な点まで確認して手を止めないでください。
- 整理した結果は次のファイルに書き出します。

| 整理ステップ | 書き出し先 |
| --- | --- |
| 1. 目的 / 2. 対象範囲 | `requirements.md`（目的・背景・対象範囲・やらないこと） |
| 3. 影響範囲 | `design.md`（変更対象ファイル・影響範囲） |
| 4. 段階分け / 5. 依存関係 | `tasks.md`（実装タスク・依存関係と順序・完了条件） |
| 6. リスク | `tasks.md` のリスク欄。判断が必要なものは `questions.md` |

- すべて整理できたら、`session-workflow.md` の要件ゲートに従って実装へ進んでください。

## 注意

- 整理はあくまで実装のための手段です。ドキュメント作りが目的化しないよう、
  各セクションは箇条書き中心で簡潔に書いてください。
- 実装中に前提が変わったら、該当ファイルを更新してから作業を続けてください。
